### AI Agent System Prompt (Updated)

```text
You are an email classification assistant. Analyze the email and provide:

CATEGORIES (exactly one):
- KIDS: Emails about children, schools, extracurriculars
- ROBYN: Personal emails for/about Robyn
- WORK: Professional/business communications
- FINANCIAL: Banking, bills, financial accounts
- SHOPPING: Orders, shipping, retail promotions
- CHURCH: Religious/church communications
- OTHER: Anything not fitting above

URGENCY (exactly one):
- HIGH: Requires immediate attention (deadlines, emergencies)
- MEDIUM: Should be addressed within 24-48 hours
- LOW: Informational, no time pressure

ACTION (exactly one):
- IGNORE: Low-value email, mark as read (promotional, newsletters)
- SHIPMENT: Contains package tracking info to extract
- DRAFT_REPLY: Requires a response, suggest draft
- JUNK: Obvious spam (ONLY for blacklisted/spam domains)
- NOTIFY: Important, send Telegram notification
- CALENDAR: Contains event/meeting/deadline date to extract

EXTRACTION:
1. For SHIPMENT emails, extract:
   - tracking_number: The shipment tracking ID
   - carrier: USPS, UPS, FedEx, Amazon, DHL, or Other
   - items: Array of item descriptions
   - estimated_delivery: ISO 8601 date

2. For CALENDAR emails, extract:
   - event_title: Brief event title
   - event_start: ISO 8601 datetime
   - event_end: ISO 8601 datetime (or null)
   - event_location: Location if mentioned

DETECTION FLAGS:
- has_tracking_info: true if email contains tracking numbers/shipping info
- has_date_info: true if email contains specific dates/times/deadlines

CONFIDENCE: 0.0-1.0 reflecting certainty in classification

Return structured JSON with all fields.
```

### Threshold Logic (Code Node)

```javascript
// Apply Action Thresholds
const classification = $input.first().json;
const confidence = classification.confidence_score || 0;
const category = classification.category;
const hasTracking = classification.has_tracking_info;
const hasDate = classification.has_date_info;

// Get sender info for blacklist/safelist check
const senderEmail = $('Get Email').first().json.sender;
const senderDomain = senderEmail.split('@')[1];

// Protected categories (never auto-JUNK)
const protectedCategories = ['KIDS', 'ROBYN', 'FINANCIAL', 'WORK'];

// Priority categories for NOTIFY
const notifyCategories = ['KIDS', 'ROBYN', 'FINANCIAL'];

// Thresholds
const THRESHOLDS = {
  IGNORE: 0.85,
  SHIPMENT: 0.90,
  DRAFT_REPLY: 0.75,
  JUNK: 0.99,
  NOTIFY: 0.85,
  CALENDAR: 0.85
};

// Decision logic (priority order)
let finalAction = 'IGNORE';
let autoAssigned = true;
let reason = 'Default fallback';

// 1. HIGH urgency â†’ NOTIFY (highest priority)
if (classification.urgency === 'HIGH' && confidence >= THRESHOLDS.NOTIFY) {
  finalAction = 'NOTIFY';
  reason = 'HIGH urgency with sufficient confidence';
}
// 2. Priority category â†’ NOTIFY
else if (notifyCategories.includes(category) && confidence >= THRESHOLDS.NOTIFY) {
  finalAction = 'NOTIFY';
  reason = `Priority category (${category}) with sufficient confidence`;
}
// 3. Has date + priority category â†’ CALENDAR
else if (hasDate && protectedCategories.includes(category) && confidence >= THRESHOLDS.CALENDAR) {
  finalAction = 'CALENDAR';
  reason = 'Date detected in priority category';
}
// 4. Has tracking info â†’ SHIPMENT
else if (hasTracking && confidence >= THRESHOLDS.SHIPMENT) {
  finalAction = 'SHIPMENT';
  reason = 'Tracking information detected';
}
// 5. Direct question/request â†’ DRAFT_REPLY (AI suggests)
else if (classification.action_v2 === 'DRAFT_REPLY' && confidence >= THRESHOLDS.DRAFT_REPLY) {
  finalAction = 'DRAFT_REPLY';
  reason = 'Response likely needed based on content';
}
// 6. Confidence too low â†’ IGNORE as safe fallback
else if (confidence < THRESHOLDS.IGNORE) {
  finalAction = 'IGNORE';
  reason = 'Confidence below threshold, safe fallback';
}
// 7. Accept AI suggestion if meets threshold
else if (confidence >= THRESHOLDS[classification.action_v2]) {
  finalAction = classification.action_v2;
  reason = 'AI classification accepted';
}
// 8. Default IGNORE
else {
  finalAction = 'IGNORE';
  reason = 'Default fallback';
}

// JUNK requires explicit blacklist check (done in Action-Processor)
// Never auto-assign JUNK here, only in Action-Processor with full checks

return {
  ...classification,
  action_v2: finalAction,
  action_confidence: confidence,
  action_auto_assigned: autoAssigned,
  assignment_reason: reason
};
```

## Webhook Configuration

### Draft-Reply-Handler Webhook

```
URL: https://n8n.coxserver.com/webhook/draft-reply-callback
Method: POST
Authentication: Telegram Bot Token verification
```

### Telegram Bot Webhook Update

```bash
# Configure Telegram to send callback_query to n8n
curl -X POST "https://api.telegram.org/bot{BOT_TOKEN}/setWebhook" \
  -d "url=https://n8n.coxserver.com/webhook/draft-reply-callback" \
  -d "allowed_updates=[\"callback_query\"]"
```
